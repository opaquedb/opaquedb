# Commit

Write the commit message in conventional-commit form and keep it plain.

## Format

```
<type>(<optional scope>): <summary>

<optional body>
```

- Pick one `type` from the list below. Add a `(scope)` only when it sharpens the
  meaning, for example `feat(client)`. Append `!` after the type or scope for a
  breaking change, for example `feat(config)!:`.
- The summary is imperative and lowercase, no trailing period, under about 70
  characters: "add client TLS", not "Added client TLS." or "adds client TLS".
- Leave a blank line, then a body only when the change needs a why. Wrap it near
  72 columns. The body explains the reason, not a restatement of the diff.

## Types

User-facing (these reach the release notes):
- `feat` a new capability
- `fix` a bug or security fix
- `hotfix` an urgent production fix
- `perf` a performance change
- `revert` undo a prior commit
- `security` a security change that is not a plain bug fix

Housekeeping (excluded from release notes):
- `docs`, `ci`, `chore`, `test`, `style`, `refactor`, `build`

## Style

- Simple and clean. Short. Say what changed and why.
- No em dashes anywhere. Use a comma, a period, or a colon.
- No AI slop: no "comprehensive", "robust", "seamless", "leverage", "delve",
  "in order to", emoji, or filler.
- No marketing. State the change, not its greatness.
- No co-author or "Generated by" trailers. This repo sets
  `includeCoAuthoredBy: false`; do not add one back.

## Procedure

1. Run `git status` and `git diff` (and `git diff --cached`) to see what will go
   in. Stage only what belongs in this commit; do not `git add -A` blindly.
2. If the staged changes cover more than one logical change, make separate
   commits. Stage hunks with `git add -p` when one file holds unrelated edits.
3. Commit only what the user asked for. Do not commit unrelated work in the tree.
4. Do not push or create branches unless asked.

## Examples

```
feat: add a TLS-capable client channel that fails closed

The client refuses to connect without TLS material or an explicit insecure
opt-in, so a bearer token never travels over a plaintext channel.
```

```
fix: require the Admin role to Insert
```

```
docs: note the per-principal rate limit in CLAUDE.md
```
